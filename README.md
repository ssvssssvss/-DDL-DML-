**Задание 1**
Ответьте на вопрос в свободной форме.
Чем частное облако отличается от общедоступного, публичного и гибридного?

**Ответ**
1. Частное облако: 
- инфраструктура используется исключительно одной организацией. 
- Может находиться в собственном дата-центре или управляться вендором на выделенных ресурсах. 
- Обеспечивает высокий уровень контроля и безопасности.

2. Публичное облако:
- инфраструктура принадлежит провайдеру (AWS, Yandex).
- Предоставляется множеству арендаторов через интернет. 
- Ресурсы делятся между клиентами.

3. Общедоступное облако аналогично публичному облаку.

4. Гибридное облако:
- сочетание частного и публичного облаков, связанных между собой (например, через VPN). 
- Позволяет держать критичные данные в частной среде, а пиковые нагрузки выносить в публичную.

Отличие: частное облако изолировано для одного клиента, публичное — многопользовательское, гибридное объединяет оба типа.

**Задание 2**
Что обозначают: IaaS, PaaS, SaaS, CaaS, DRaaS, BaaS, DBaaS, MaaS, DaaS, NaaS, STaaS? Напишите примеры их использования.

**Ответ**
- IaaS (Infrastructure as a Service  — предоставление виртуальной инфраструктуры (серверы, сети, диски).
Пример: аренда виртуальной машины в Yandex Compute Cloud или AWS EC2.

- PaaS (Platform as a Service) — предоставление платформы для разработки и развертывания приложений без управления ОС и инфраструктурой.
Пример: Yandex Cloud Managed Service for Kubernetes, AWS Elastic Beanstalk.

- SaaS (Software as a Service) — готовое приложение, доступное через интернет.
Пример: Yandex 360, Google Docs, Salesforce.

- CaaS (Containers as a Service) — платформа для управления контейнерами (оркестрация).
Пример: Yandex Managed Service for Kubernetes, Amazon EKS.

- DRaaS (Disaster Recovery as a Service) — услуга аварийного восстановления, репликация и резервное копирование инфраструктуры на случай сбоя.
Пример: AWS CloudEndure Disaster Recovery.

- BaaS (Backup as a Service) — услуга резервного копирования данных в облако.
Пример: Yandex Cloud Backup, AWS Backup.

- DBaaS (Database as a Service) — управляемая база данных, провайдер берет на себя администрирование, резервное копирование и масштабирование.
Пример: Yandex Managed Service for PostgreSQL, Amazon RDS.

- MaaS (Monitoring as a Service) — облачный мониторинг инфраструктуры и приложений.
Пример: Yandex Cloud Monitoring, AWS CloudWatch.

- DaaS (Desktop as a Service) — виртуальные рабочие столы, доступные удаленно.
Пример: Amazon WorkSpaces.

- NaaS (Network as a Service) — предоставление сетевых сервисов (VPN, балансировщики, маршрутизация) как услуги.
Пример: Yandex Virtual Private Cloud, AWS VPC.

- STaaS (Storage as a Service) — облачное хранилище данных.
Пример: Yandex Object Storage, AWS S3.

**Задание 3**
Ответьте на вопрос в свободной форме.
Напишите, какой вид сервиса предоставляется пользователю в ситуациях:
Всеми процессами управляет провайдер.
Вы управляете приложением и данными, остальным управляет провайдер.
Вы управляете операционной системой, средой исполнения, данными, приложениями, остальными управляет провайдер.
Вы управляете сетью, хранилищами, серверами, виртуализацией, операционной системой, средой исполнения, данными, приложениями.

**Ответ**
- Всеми процессами управляет провайдер — SaaS (Software as a Service). Пользователь только использует готовое приложение, не управляя ни инфраструктурой, ни платформой.
- Вы управляете приложением и данными, остальным управляет провайдер — PaaS (Platform as a Service). Пользователь развертывает свои приложения и работает с данными, а провайдер управляет инфраструктурой, ОС, middleware и средой исполнения.
- Вы управляете операционной системой, средой исполнения, данными, приложениями, остальным управляет провайдер — IaaS (Infrastructure as a Service). Пользователь получает виртуальные ресурсы (серверы, сети, диски) и самостоятельно настраивает ОС и вышележащие компоненты, провайдер управляет физической инфраструктурой и гипервизором.
- Вы управляете сетью, хранилищами, серверами, виртуализацией, операционной системой, средой исполнения, данными, приложениями — On-premise (локальная инфраструктура) или частное облако в модели «голое железо» (bare-metal). Пользователь самостоятельно управляет всеми уровнями — от оборудования до приложений.

Какие ОС можно выбрать?
- Ubuntu'
- Debian OS logun
- Ollama + WebUI
- CentOS
- Fedora
- и другие из Marketplace

Какие параметры сервера можно выбрать?
- Платформа (процессор): Intel Broadwell, Intel Cascade Lake, Intel Ice Lake, AMD EPYC
- Вычислительные ресурсы: Количество vCPU (от 2 до 64 и более), гарантированная доля vCPU (5%, 20%, 50%, 100%), объем RAM (от 1 ГБ до 256 ГБ и более)
- Диски: HDD (network-hdd), SSD (network-ssd), Non-replicated SSD (network-ssd-nonreplicated), Ultra SSD. Размер: от 3 ГБ до нескольких ТБ
- Сеть: подсеть, публичный IP-адрес (автоматически или статический), группы безопасности (файрвол)
- Доступ: Логин, SSH-ключ (для Linux), пароль (для Windows), сервисный аккаунт
- Дополнительно: прерываемость (снижает стоимость), агент сбора метрик, резервное копирование

Какие компоненты мониторинга можно создать?
- Агент сбора метрик.
- Размещение. Пул резервов ВМ: зарезервированная в заданной зоне доступности совокупность вычислительных ресурсов, которые будут гарантированно доступны пользователю для создания в этой зоне доступности виртуальных машин в определенной конфигурации. 

Какие системы безопасности предусмотрены?
- Сетевой уровень: Группы безопасности (Security Groups) — облачный файрвол с правилами входящего/исходящего трафика. Поддержка сетевых политик на уровне подсетей и ВМ.
- Управление доступом: IAM (Identity and Access Management) — ролевая модель доступа. Сервисные аккаунты с ограниченными правами. MFA (многофакторная аутентификация).
- Шифрование: KMS (Key Management Service) — управление ключами шифрования. Шифрование дисков и объектов в Object Storage. TLS для передачи данных.
- Резервное копирование: Cloud Backup — создание резервных копий ВМ. Версионирование в Object Storage для защиты от случайного удаления.
- Безопасность ВМ: SSH-ключи вместо паролей. Отключение последовательной консоли по умолчанию. Возможность использования выделенных хостов (Dedicated Hosts) для изоляции.
- Мониторинг безопасности: Cloud Audit Logs — аудит всех действий с ресурсами. Обнаружение аномалий через Monitoring.
- Защита приложений: Marketplace-решения: Webmonitorx (WAF-защита веб-приложений), CloudGuard.

Как меняется цена от выбранных характеристик? Приведите пример самой дорогой и самой дешёвой конфигурации.
- Вычислительные ресурсы: количество vCPU, гарантированная доля (5%, 20%, 50%, 100%), объем RAM.
- Тип ВМ: обычная или прерываемая (дешевле на 50-70%).
- Диски: тип (HDD/SSD) и размер.
- ОС: использование Windows увеличивает стоимость (плата за лицензию).
- Сеть: публичный IP-адрес и исходящий трафик.
- Платформа процессора: более новые платформы (Intel Ice Lake) могут иметь иную стоимость.
- Тарификация: посекундная, ВМ в статусе STOPPED не тарифицируется (диски продолжают оплачиваться).
